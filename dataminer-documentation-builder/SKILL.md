# Documentation Builder Skill

## Purpose

Generate a complete documentation website for a DataMiner SDM solution and package
it for deployment. This is a two-stage pipeline:

1. **DocFX Builder** — scaffolds the site, fills in content, builds `_site`
2. **Documentation Installer** — packages the built site into a `.dmapp`

---

## Prerequisites

- **DocFX** installed: `dotnet tool install -g docfx`
- **Input YAML** — the domain model definition file (from Model Analyzer)
- **Solution artifacts** — DevPack built, UDAPI built, SolutionDescription.md available

---

## Pipeline Steps

### Step 1: Run DocFX Builder

Scaffolds the documentation site structure with Skyline branding, placeholder
pages, and `docfx.json` configuration.

```powershell
cd dataminer-documentation-builder/dataminer-docfx-builder
dotnet run New-DocfxBuilder.cs -- --input-yaml <path> --output-dir <root>
```

**Output:** `<OutputDir>/<SolutionName>Documentation/` with all scaffolded files.

See [dataminer-docfx-builder/SKILL.md](dataminer-docfx-builder/SKILL.md) for
content-filling instructions after scaffolding.

### Step 2: Fill Documentation Content

After scaffolding, replace all `<!-- TODO -->` placeholders with real content
derived from:

| Source File | Used For |
|-------------|----------|
| `SolutionDescription.md` | Landing page, architecture overview, getting started |
| Input YAML | Model documentation, enum references, relationship diagrams |
| `openapi.yaml` | UDAPI endpoint reference, request/response examples |
| DevPack XML docs | API reference (auto-generated by docfx metadata) |

### Step 3: Extract API Metadata and Build the DocFX Site

```powershell
cd <OutputDir>/<SolutionName>Documentation

# Extract C# API metadata from devpack project (MANDATORY — do NOT skip)
docfx metadata docfx.json

# Build the static HTML site
docfx build docfx.json
```

**CRITICAL**: `docfx metadata` must run before `docfx build`. Without it, the
API Reference section (`devpack/api/`) will be empty and show 404 errors.

**Output:** `_site/` folder with the static HTML website including API reference pages.

### Step 4: Run Documentation Installer

Packages the built `_site` into a `.dmapp` for DataMiner deployment.

```powershell
cd dataminer-documentation-builder/dataminer-documentation-installer
dotnet run New-DocumentationInstaller.cs -- --input-yaml <path> --output-dir <root>
```

**Output:** `<SolutionName>Documentation/<SolutionName>Documentation.Package/` with
the built `.dmapp` file.

---

## Quick Run (All Steps)

For a complete documentation build from scratch:

```powershell
# 1. Scaffold
dotnet run dataminer-docfx-builder/New-DocfxBuilder.cs -- -i <yaml> -o <dir>

# 2. Fill content (agent-driven — see docfx-builder SKILL.md)

# 3. Extract API metadata + build site
cd <dir>/<Name>Documentation && docfx metadata docfx.json && docfx build docfx.json && cd -

# 4. Package for DataMiner
dotnet run dataminer-documentation-installer/New-DocumentationInstaller.cs -- -i <yaml> -o <dir>
```

## Output Structure

```
<OutputDir>/
└── <SolutionName>Documentation/
    ├── docfx.json
    ├── toc.yml
    ├── index.md
    ├── articles/
    │   ├── toc.yml
    │   ├── getting-started.md
    │   └── architecture.md
    ├── devpack/
    │   ├── toc.yml
    │   ├── index.md
    │   └── api/                    (auto-generated from XML docs)
    ├── webapi/
    │   ├── toc.yml
    │   ├── index.md
    │   ├── endpoints.md
    │   └── examples.md
    ├── templates/skyline/          (Skyline branded theme)
    ├── images/
    ├── _site/                      (built static site)
    └── <SolutionName>Documentation.Package/
        ├── <Name>.csproj
        └── PackageContent/
            └── CompanionFiles/
                └── Skyline DataMiner/
                    └── Webpages/Public/Documentation/<SolutionName>/
```

---

## Deployment

After installation on DataMiner, the documentation is accessible at:

```
http://<dataminer-host>/public/Documentation/<SolutionName>/index.html
```

---

## Re-running After Changes

If documentation source files change:

1. Re-run the DocFX Builder (or just edit markdown files directly)
2. Rebuild: `docfx metadata docfx.json && docfx build docfx.json`
3. Re-run the Documentation Installer — it always re-copies the latest `_site`
