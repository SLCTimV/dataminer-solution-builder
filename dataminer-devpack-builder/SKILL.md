# Devpack Builder Skill

## Purpose

Generate the complete DataMiner SDM backend devpack from a YAML domain model
definition. This includes solution scaffolding, C# model/enum/sub-object classes,
API helpers, DomMapper generation (via the SDM CLI tool), DOM installer code, the
package installer entry point, and a final NuGet build.

## Script

```
New-DevPack.cs
```

A single-file .NET 10 program. No `.csproj` required — run directly with:

```powershell
dotnet run New-DevPack.cs -- --input-yaml <path-to-yaml> [--output-dir <root>]
```

### Parameters

| Parameter          | Required | Default    | Description |
|--------------------|----------|------------|--------------|
| `-i, --input-yaml` | Yes      | —          | Path to the YAML domain model definition file |
| `-o, --output-dir` | No       | `C:\temp`  | Root directory where solution folders are created |

## Steps Performed

| Step | Description |
|------|-------------|
| 1/8 | Scaffold domain library — solution file, NuGet project, package references |
| 2/8 | Scaffold package project — .dmapp installer project, package references |
| 3/8 | Generate model C# files — main models (`SdmObject<T>`), enums, sub-objects |
| 4/8 | Generate API helpers — `I{Model}ApiHelper` interface + `{Model}ApiHelper` implementation |
| 5/8 | Run SDM DomMapper tool — generates `*DomMapper.g.cs`, copies to Package/DOM |
| 6/8 | Generate DOM installer — partial `DomInstaller` classes with section/definition builders |
| 7/8 | Generate Package.cs — installer entry point with SDM prerequisite check |
| 8/8 | Build NuGet package — compiles the domain library |

## YAML Input Format

The YAML must have at minimum a `solution` block. Supports both single-model
(`mainModel`) and multi-model (`models` array) schemas.

```yaml
solution:
  name: "SDMEvent"                              # used as solution/project name prefix
  domModuleId: "exampleeventmgmt"
  nugetPackageId: "Skyline.DataMiner.Utils.SDMEvent"
  apiRoute: "eventmanager/events"
  apiName: "Example Event Manager API"
  apiDescription: "The webapi for the example event manager."

mainModel:                                      # OR use 'models:' for multi-model
  name: "Event"
  properties:
    - { name: "Name", type: "string" }
    - { name: "Status", type: "enum", enum: "EventStatus" }
    - { name: "Owner", type: "ref", ref: "User" }
  lists:
    - { name: "Languages", type: "Language" }

enums:
  - name: "EventStatus"
    values: ["Requested", "Processing", "Done"]

subObjects:
  - name: "Language"
    properties:
      - { name: "Name", type: "string" }
      - { name: "AudioType", type: "enum", enum: "AudioType" }
```

### Supported property types

| Type | C# mapping | DOM storage |
|------|-----------|-------------|
| `string` | `string` | `FieldDescriptor` (typeof string) |
| `DateTime` | `DateTime` | `FieldDescriptor` (typeof DateTime) |
| `int` | `int` | `FieldDescriptor` (typeof int) |
| `double` | `double` | `FieldDescriptor` (typeof double) |
| `bool` | `bool` | `FieldDescriptor` (typeof bool) |
| `enum` | enum class name | `GenericEnumFieldDescriptor` (int-backed) |
| `ref` | `Guid` (property named `{Name}Id`) | `FieldDescriptor` (typeof string) |

## Output Structure

```
<OutputDir>/
└── SDM<Name>/
    ├── SDM<Name>.sln
    ├── SDM<Name>/                       ← Utils.<Name>.csproj
    │   ├── Models/
    │   │   ├── <Model>.cs               ← SdmObject<T> with [SdmDomStorage], [GenerateExposers]
    │   │   ├── <Enum>.cs                ← C# enum
    │   │   ├── <SubObject>.cs           ← Plain POCO class
    │   │   └── <Model>DomMapper.g.cs    ← Generated by SDM CLI tool
    │   └── ApiHelpers/
    │       ├── I<Model>ApiHelper.cs     ← Interface with IBulkRepository<T>
    │       └── <Model>ApiHelper.cs      ← Implementation using DomRepository
    └── SDM<Name>.Package/               ← .dmapp installer project
        ├── SDM<Name>.Package.cs         ← Install entry point
        └── DOM/
            ├── DomInstaller.cs          ← Module + InstallDefaultContent()
            ├── <Model>.cs               ← Partial class with section/definition installers
            └── <Model>DomMapper.g.cs    ← Copy from Models/
```

## NuGet Dependencies (always latest)

### Domain library (`Utils.SDM<Name>`)
- `Skyline.DataMiner.Dev.Common`
- `Skyline.DataMiner.SDM`

### Package project (`SDM<Name>.Package`)
- `Skyline.DataMiner.Dev.Automation`
- `Skyline.DataMiner.Utils.DOM`
- `Skyline.DataMiner.SDM.SourceGenerator.Runtime`
- `Skyline.DataMiner.Utils.SecureCoding`

> **NuGet version pinning**: all packages are currently resolved to their latest
> stable version at scaffold time. To pin versions, pass the version explicitly
> via `dotnet add package <id> --version <ver>` or add a `Directory.Packages.props`
> file with `<PackageVersion>` entries.

## Prerequisites

- .NET 10 SDK
- DataMiner dotnet templates installed:
  ```powershell
  dotnet new install Skyline.DataMiner.VisualStudioTemplates
  ```
- SDM CLI tool (auto-installed if missing):
  ```powershell
  dotnet tool install skyline.dataminer.sdm.tools --global
  ```
- `YamlDotNet` NuGet package (declared via `#:package` directive, restored automatically on first run)

## Pipeline Position

```
Stage 1 — Business Model Extractor  →  YAML definition
Stage 2 — Devpack Builder (THIS)    →  Full backend: solution + models + helpers + DOM installer + NuGet
Stage 3 — UDAPI Generator           →  Controllers, extensions, openapi.yaml
Stage 4 — Deploy                    →  Upload .dmapp to DataMiner agent
```
